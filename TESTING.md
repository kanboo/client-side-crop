# 測試指南

## 啟動開發伺服器

```bash
pnpm dev
```

開啟瀏覽器訪問 `http://localhost:5173`

## 功能測試清單

### ✅ 1. 圖片選擇功能

- [ ] 點擊「選擇圖片」按鈕
- [ ] 選擇一張圖片（JPEG、PNG、GIF、WebP 或 BMP）
- [ ] 確認圖片正確載入到裁切區

### ✅ 2. 檔案驗證

**格式驗證：**

- [ ] 嘗試上傳不支援的格式（如 .txt、.pdf）
- [ ] 應顯示錯誤訊息：「不支援的圖片格式...」

**大小驗證：**

- [ ] 嘗試上傳超過 10MB 的圖片
- [ ] 應顯示錯誤訊息：「圖片大小超過 10MB 限制」

### ✅ 3. 裁切功能

**移動裁切框：**

- [ ] 拖曳裁切框中央區域
- [ ] 確認可以自由移動位置

**調整大小：**

- [ ] 拖曳裁切框的四個角落控制點
- [ ] 拖曳裁切框的四個邊緣控制點
- [ ] 確認比例維持在 9:16（寬:高）

**縮放：**

- [ ] 使用滑鼠滾輪或觸控板縮放（如有設定）
- [ ] 確認裁切框可放大縮小

### ✅ 4. 即時預覽

- [ ] 移動或調整裁切框時
- [ ] 右側預覽區應即時更新
- [ ] 預覽保持 9:16 比例

### ✅ 5. 裁切資訊顯示

- [ ] 調整裁切框大小時
- [ ] 確認下方顯示的寬高數值即時更新
- [ ] 格式：「📐 裁切資訊：寬 XXX px × 高 XXX px」

### ✅ 6. 重設功能

- [ ] 調整裁切框後點擊「重設」按鈕
- [ ] 確認裁切框回到初始位置和大小（70% 覆蓋）

### ✅ 7. 取消功能

- [ ] 載入圖片後點擊「取消」按鈕
- [ ] 確認圖片被清除，回到初始狀態

### ✅ 8. 下載功能

- [ ] 調整裁切框到想要的區域
- [ ] 點擊「下載」按鈕
- [ ] 確認瀏覽器開始下載裁切後的圖片
- [ ] 開啟下載的圖片，確認：
  - 裁切區域正確
  - 保持原圖片格式（JPEG/PNG/等）
  - 比例為 9:16

### ✅ 9. 上傳功能

- [ ] 調整裁切框到想要的區域
- [ ] 開啟瀏覽器開發者工具的 Console
- [ ] 點擊「上傳」按鈕
- [ ] 確認 Console 顯示：
  ```
  上傳檔案： File {...}
  檔案名稱： cropped-xxx.jpg
  檔案大小： XX.XX KB
  檔案類型： image/jpeg
  ```

### ✅ 10. 響應式設計

- [ ] 調整瀏覽器視窗大小
- [ ] 在小螢幕（< 768px）時，預覽區應移到裁切區下方
- [ ] 確認在各種尺寸下都能正常操作

## 瀏覽器相容性測試

建議在以下瀏覽器測試：

- [ ] Chrome/Edge（最新版）
- [ ] Firefox（最新版）
- [ ] Safari（最新版，macOS/iOS）

## 效能測試

- [ ] 上傳大尺寸圖片（接近 10MB）
- [ ] 確認裁切操作流暢，無明顯延遲
- [ ] 下載/上傳轉換速度合理（< 2 秒）

## 已知限制

1. GIF 動畫裁切後只保留第一幀（這是預期行為）
2. 不支援 HEIC/HEIF 格式（iPhone 原生格式，需轉換）
3. 不支援 AVIF 格式（舊版瀏覽器不支援）

## 測試完成後

如果所有功能都正常運作，表示實作成功！🎉

如有任何問題，請檢查：

1. 瀏覽器 Console 是否有錯誤訊息
2. 網路請求是否正常（開發者工具 Network 頁籤）
3. TypeScript 型別檢查：`pnpm type-check`
4. ESLint 檢查：`pnpm lint`
